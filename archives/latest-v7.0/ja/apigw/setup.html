2. 開発環境のセットアップ¶

2.1. コマンドライン開発環境のセットアップ¶

API Gateway 定義を BaaS サーバに登録するには以下の２つの方法があります。

  • ブラウザを利用してデベロッパーコンソール上で Swagger 定義を投入
  • コマンドラインツール(CLI)を使用して Swagger 定義を BaaS サーバに投入

CLI を使う場合には CLI のインストールおよびプロジェクトディレクトリの作成が必要です。 デベロッパーコンソールから投入する場合は以下の手順は不要です。

2.1.1. CLI のインストール¶

API Gateway の定義をコマンドラインから登録・変更するためには、Nebula CLI のインストールが必要になります。

Nebula CLI はCloud Functions開発で使用するものと同じです。 インストール手順はCloud Functions開発ガイドの 開発環境のセットアップ を参照してください。

なお Nebula CLI のリファレンスは CLIツール コマンドリファレンス を参照してください。

2.1.2. プロジェクトディレクトリの作成¶

API Gateway 定義を行うにあたり、プロジェクトディレクトリの作成と BaaS サーバへの接続設定を行う必要があります。

手順については Cloud Functions開発ガイド の 「プロジェクトディレクトリの作成」を参照してください。
