13. 注意事項¶

13.1. HTTPタイムアウトについて¶

.NET SDK では HTTP の通信タイムアウトをデフォルトで無限大に設定しています。 これは大きな容量のファイルデータを低速回線でアップロード・ダウンロードする際にも、 エラーが発生しないようにするためです。

逆に、HTTP リクエスト実行中に通信経路上でネットワーク切断が発生した場合でも、 タイムアウトによるエラー通知が行われない場合があります (TCP再送タイムアウトの場合は通知されます)。

なお、タイムアウト値は変更可能です。詳細は HTTPリクエストタイムアウト・キャンセル を参照してください。
