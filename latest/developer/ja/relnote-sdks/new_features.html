新機能¶

本リリースはバージョン ver 7.5.0 です。 本バージョンでは、以前リリースしたバージョンからいくつかの機能拡張・変更がなされています。

ver 7.0.x → ver 7.5.0 での新機能は以下のとおりです。 なお、サーバ側の機能強化については「NECモバイルバックエンド基盤 サーバ マニュアル」の「リリースノート」を参照ください。

注釈

本リリースは、BaaS サーバ ver 7.5 以上に対応しています。 BaaS サーバ ver 7.0 以下との接続は保証しておりませんのでご注意ください。

注意

「β版」と記載している機能は AS IS で提供するものです。これら機能については現時点で品質保証はしておりませんのでご注意ください。

Python SDK¶

Python に対応した Python SDK を追加しました。

HTTP/2対応¶

HTTP/2 プロトコルに対応しました。従来の HTTP/1.1 よりも効率的に REST API 通信を行うことができます。 クライアント SDK のうち、HTTP/2 に対応したのは以下の SDK です。

  • Java/Android SDK
  • JavaScript SDK (Node.jsのみ)

JSONオブジェクトストレージ¶

JSONフィールド暗号化¶

格納する JSON の特定フィールドを暗号化して保存ができるようになりました。

詳細は データ暗号化 を参照ください。

トランザクション¶

MongoDB 4.0 のトランザクション機能を使用することができるようになりました。

詳細は トランザクション を参照ください。

ACLレスバケット¶

オブジェクト単位の ACL 制御を省略する機能を追加しました。 オブジェクト単位でのアクセス制御ができませんが、ACLが付与されないのでデータサイズを圧縮できます。

詳細は ACLレスバケット を参照してください。

注意

ACLレスバケット機能はβ版扱いとなります。

Java/Android SDK¶

オブジェクトストレージのクエリ時に Projection の利用が可能になりました。

ファイルストレージ¶

部分ダウンロード¶

Rangeヘッダによる範囲指定での部分ファイルダウンロード機能を追加しました。 クライアント SDK のうち、部分ファイルダウンロードに対応したのは以下の SDK です。

  • JavaScript SDK (Node.jsのみ)

Cloud Functions¶

イベント駆動¶

ファイル登録などのイベントに連動して Function を起動する機能を追加しました。

詳細は イベント駆動 開発ガイド を参照してください。

ユーザ管理¶

  • ユーザ管理機能(登録、更新、削除)のバッチ処理APIを追加しました。
  • ユーザを無効化する機能を追加しました。

詳細は REST API リファレンス: 本編 ⇒ ユーザ管理 を参照してください。

管理 REST API¶

テナント管理、アプリ管理などの管理用 REST API を追加しました。

詳細は REST API リファレンス: 管理API編 を参照してください。

認証¶

REST API の認証に Basic認証が使えるようになりました。

詳細は 認証機能ガイド ⇒ ローカル認証: Basic認証 を参照してください。
