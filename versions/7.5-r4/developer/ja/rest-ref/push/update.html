10.3. インスタレーションの更新¶

PUT /1/(tenantId)/push/installations/(installationId)¶

インスタレーションを新規登録・再登録する。

Request Headers:
 
  • X-Application-Id -- アプリケーションID(必須)
  • X-Application-Key -- アプリケーションキー(必須)
  • X-Session-Token -- セッショントークン(オプション)
  • Content-Type -- "application/json"
Parameters:
  • installationId (string) -- インスタレーションID
Status Codes:
  • 200 OK -- 正常更新
  • 400 Bad Request -- パラメータ・リクエストボディ不正
  • 401 Unauthorized -- 認証エラー
  • 404 Not Found -- 該当するインスタレーションが存在しない
  • 415 Unsupported Media Type -- Content-Type 不正

リクエストボディ

更新用の JSON オブジェクト。フォーマットは オブジェクトの更新 と同じ。

レスポンスボディ

更新後のインスタレーション情報がJSON形式で返却される。

注意事項

  • インスタレーションを部分更新で想定外のフォーマットに変更してしまった場合、インスタレーションは自動削除され、404 Not Found が返却される。
