9.1. テナント所属管理者登録・更新¶

PUT /1/_sysadm/(tenantId)/tenantDevelopers¶

テナントに所属する管理者を登録・更新する

Request Headers:
 
  • X-Developer-Token -- 管理者用セッショントークン（必須）
  • Content-Type -- application/json, application/yaml のいずれか
Parameters:
  • tenantId (string) -- テナントID
Request JSON Object:
 
  • tenantDeveloper.email (string) -- E-Mail（必須）
  • tenantDeveloper.accountType (string) -- アカウント種別(必須)
Status Codes:
  • 200 OK -- 正常登録した
  • 400 Bad Request -- リクエストボディ不正
  • 401 Unauthorized -- 認証エラー
  • 403 Forbidden -- 権限エラー
  • 404 Not Found -- 該当テナントが存在しない、または無効。該当メールアドレスが存在しない
  • 415 Unsupported Media Type -- Content-Type 不正
Response JSON Object:
 
  • tenantDeveloper.email (string) -- E-Mail
  • tenantDeveloper.accountType (string) -- アカウント種別

リクエストボディ

YAML 形式によるリクエストの例を以下に示す。

tenantDeveloper:
  email: example1@test.com
  accountType: appAdmin

アカウント種別は、以下いずれかを指定する。

  • tenantAdmin : テナント管理者
  • appAdmin : アプリケーション管理者
  • opAdmin : 運用管理者

注意事項

  • 管理者未登録のメールアドレスは指定不可。
